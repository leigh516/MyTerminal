# Product Requirement Document (PRD)
## 프로젝트명: Terminal Avy (터미널 에이비)

본 문서는 Windows 환경에서 구동되는 고성능, 경량 AI 통합 터미널 어플리케이션 **'Terminal Avy'**의 제품 요구사항 정의서(PRD)입니다. 본 프로그램은 Warp 스타일의 블록형 터미널, 원활한 로컬-SSH 간 파일 드래그 앤 드롭 전송, 내장 파일 탐색기 및 코드 에디터, 그리고 로컬 LLM 연동 AI 채팅 인터페이스를 통합 제공하는 것을 목표로 합니다.

---

## 1. 제품 개요 (Product Overview)
* **목적**: 기존 터미널의 한계를 넘어 개발자가 단일 윈도우 환경에서 터미널 작업, 파일 탐색, 코드 편집 및 AI 코파일럿의 도움을 유기적으로 받을 수 있는 차세대 개발자 워크스페이스 구축
* **핵심 가치**:
  1. **생산성 극대화**: 터미널 명령어 단위 블록화 및 원클릭 복사/재사용
  2. **유기적 파일 관리**: SSH와 로컬 세션 간의 마우스 드래그 앤 드롭 파일 복사/이동
  3. **AI 워크플로우 통합**: 로컬 LLM 연동을 통해 보안 걱정 없이 터미널 내부에서 AI 질의 및 명령어 실행 요청 가능
  4. **최상의 한글 지원**: 한글 입력기(IME) 버그 없는 자연스러운 타이핑 및 한글 폰트 최적화

---

## 2. 타겟 유저 및 유즈케이스 (Target Audience & Use Cases)
* **타겟 유저**:
  * Windows 환경에서 개발 및 인프라 운영을 수행하는 소프트웨어 엔지니어, DevOps 엔지니어, 시스템 관리자
  * 로컬 개발과 원격 SSH 서버 관리를 병행하며 잦은 파일 전송이 필요한 개발자
  * ChatGPT, Ollama 등 AI 도구를 터미널 작업에 적극적으로 활용하고자 하는 개발자

* **주요 유즈케이스**:
  * **명령어 블록 재사용**: 터미널에서 실행했던 복잡한 명령어나 그 출력 결과를 블록 단위로 복사하여 동료에게 공유하거나 스크립트 작성 시 재사용
  * **원격-로컬 파일 전송**: SSH 터미널 세션과 로컬 탐색기 간에 마우스 드래그만으로 설정 파일이나 빌드 결과물을 전송
  * **코드 즉시 편집**: 파일 트리에서 소스코드를 더블클릭하여 터미널 내장 에디터로 수정하고 즉시 터미널에서 컴파일/실행
  * **AI 명령어 실행**: 빌드 오류나 에러 로그가 발생했을 때 하단 AI 채팅창에 도움을 요청하고, AI가 제안한 명령어를 원클릭으로 터미널에 실행

## 3. 핵심 요구사항 및 기능 명세 (Feature Specifications)

### 3.1. Warp 스타일 블록형 터미널 (Warp-style Block Terminal)
* **요구사항 1**: 입력한 명령어와 그에 따른 실행 결과 출력(Stdout/Stderr)이 하나의 **'UI 블록'**으로 묶여 시각적으로 구분되어야 합니다.
* **요구사항 2**: 각 블록 우측 상단에 퀵 액션 버튼(블록 명령어 복사, 출력 결과 전체 복사, 블록 재생성/재실행, 즐겨찾기 등록)을 제공해야 합니다.
* **요구사항 3**: 긴 출력 결과가 발생하는 블록의 경우 자체적으로 접기/펼치기(Collapse/Expand) 기능이 제공되어 스크롤 낭비를 방지해야 합니다.

### 3.2. 드래그 앤 드롭 파일 탐색 및 전송 (Drag & Drop File Explorer & Transfer)
* **요구사항 1 (내장 파일 트리)**: 터미널 사이드바에 현재 로컬 경로 또는 연결된 SSH 세션의 파일 시스템 구조를 나타내는 파일 트리를 제공해야 합니다.
* **요구사항 2 (드래그 앤 드롭 전송)**:
  * 로컬 탐색기(Windows Explorer)에서 파일을 드래그하여 터미널 SSH 세션 화면이나 내장 SSH 파일 트리에 드롭하면, 해당 SSH 경로로 파일이 업로드(SCP/SFTP 프로토콜 활용)되어야 합니다.
  * 반대로 SSH 파일 트리에서 파일을 드래그하여 로컬 터미널 세션이나 윈도우 탐색기로 드롭하면 다운로드되어야 합니다.
* **요구사항 3 (진행률 표시)**: 대용량 파일 복사/이동 시 전송 속도 및 진행 상태를 나타내는 진행 바(Progress Bar)가 오버레이 형태로 제공되어야 합니다.

### 3.3. 다중 세션 및 자유로운 화면 분할 (Session & Split View Management)
* **요구사항 1 (화면 분할)**: 사용자가 원하는 대로 가로/세로 방향으로 무한히 화면을 분할(Split Screen)할 수 있어야 합니다.
* **요구사항 2 (독립적 폰트 스케일링)**: 각 분할된 세션별로 포커스된 상태에서 `Ctrl + '+'` 및 `Ctrl + '-'` 단축키를 입력 시 해당 세션만 독자적으로 폰트 크기가 조정되어야 합니다.
* **요구사항 3 (세션 및 레이아웃 저장/불러오기)**:
  * 현재 영구 저장되어 있는 세션 연결 상태(로컬 쉘 종류, SSH 접속 정보 등) 및 화면 분할 레이아웃 구성을 저장할 수 있어야 합니다.
  * 저장 시 사용자 지정을 통한 세션 그룹명 및 타이틀 설정이 가능해야 하며, 추후 해당 프로필을 불러오면 이전에 구성된 세션들이 한 번에 재복구되어야 합니다.

### 3.4. 파일 트리 내장 에디터 (Integrated Code Editor)
* **요구사항 1**: 파일 트리에서 특정 텍스트 기반 소스코드 파일(.txt, .json, .py, .js, .cpp 등)을 선택하거나 더블클릭하면, 터미널 윈도우 내부 혹은 분할 뷰의 한 영역으로 편집 가능한 코드 에디터 화면이 실행되어야 합니다.
* **요구사항 2**: 기본적인 코드 하이라이팅, 자동 완성, 줄 번호 표시가 지원되어야 합니다 (Lightweight Monaco Editor 또는 CodeMirror 라이브러리 검토).
* **요구사항 3**: `Ctrl + S` 저장 시 로컬 또는 원격 SSH 파일 시스템에 즉시 동기화 반영되어야 합니다.

### 3.5. 한글 입력 및 렌더링 최적화 (Korean Language Optimization)
* **요구사항 1 (IME 최적화)**: 터미널 명령어 입력 창 및 에디터에서 한국어 조합 중 자음/모음이 분리되거나 마지막 글자가 누락되는 현상이 전혀 없어야 합니다.
* **요구사항 2 (한글 폰트 지원)**: D2Coding, 나눔고딕 코딩 등 한글 개발용 폰트가 기본 지정 또는 손쉽게 적용 가능해야 하며, 한글과 영문 가독성이 균형 잡힌 글자폭(Double-width character) 정렬이 깨지지 않아야 합니다.

### 3.6. 고정 AI 채팅창 및 로컬 LLM 연동 (AI Chat Panel & Local LLM Integration)
* **요구사항 1 (하단 AI 채팅창)**: 애플리케이션 하단에 상시 고정할 수 있는 채팅 입력창 UI를 제공하여 AI와 대화할 수 있어야 합니다.
* **요구사항 2 (맥락 공유)**: 사용자가 현재 겪고 있는 터미널 에러 로그를 블록 선택하여 'AI에게 질문'을 누르면 자동으로 하단 채팅 입력창에 컨텍스트로 삽입되어야 합니다.
* **요구사항 3 (LLM 설정 메뉴)**:
  * 사용자가 API 주소(Endpoint URL), API Key(필요한 경우), 사용 모델명(Model Name), 시스템 프롬프트 등을 직접 커스텀할 수 있는 설정 페이지를 제공해야 합니다.
  * 특히 오프라인 환경에서도 작동할 수 있도록 로컬 LLM 서버(Ollama, LM Studio 등 Local API)와의 연동(OpenAI API 호환 규격)을 지원해야 합니다.

## 4. 비기능적 요구사항 (Non-Functional Requirements)
* **성능 및 리소스 소비 (Lightweight)**:
  * 기존 Electron 기반의 무거운 앱들(예: VS Code, Warp)에 비해 메모리 점유율을 대폭 낮추고 실행 속도를 빠르게 하기 위해 백엔드 코어는 가벼운 네이티브 언어(Rust, C++ 등) 기반으로 설계하는 것을 강력히 권장합니다.
  * 메모리 릭(Memory Leak)이 없어야 하며, 수천 줄의 명령어 결과가 출력되어도 스크롤 버벅임이 없는 고성능 렌더링 기술(GPU 가속화 렌더링 등)이 적용되어야 합니다.
* **보안**: 
  * SSH 자격 증명(비밀번호, Private Key 등)은 Windows Credential Manager 또는 안전한 암호화 저장소에 보관되어야 합니다.
  * AI 통신 시 로컬 LLM을 기본으로 고려하여 외부로 소스코드 및 보안 데이터가 유출되지 않도록 구조적으로 제한해야 합니다.
* **오류 관리**: 
  * 비정상적인 종료나 세션 끊김 시 마지막 세션 상태가 자동으로 임시 저장되어 유실을 방지해야 합니다.

## 5. 기술 아키텍처 및 추천 기술 스택 (Technical Architecture)

| 구분 | 추천 스택 | 비고 / 장점 |
| :--- | :--- | :--- |
| **데스크톱 프레임워크** | **Tauri (Rust + Frontend)** | Electron 대비 매우 가볍고(빌드 본체 용량 ~10MB 내외), 메모리 소모가 최소화됨. Rust 백엔드로 SSH 프로토콜 및 로컬 파일 탐색기를 고성능으로 제어 가능. |
| **프론트엔드 UI/UX** | **React.js + Tailwind CSS** | 풍부한 UI 라이브러리 생태계 활용 가능, 반응형 웹 디자인 용이 |
| **터미널 에뮬레이터** | **xterm.js** | VS Code 등에서 검증된 고성능 터미널 컴포넌트, WebGL 애드온을 통한 GPU 가속화 지원, 한글 정렬 및 스케일링 지원 우수 |
| **내장 코드 에디터** | **Monaco Editor** or **CodeMirror** | 가벼우면서도 구문 강조 및 자동완성이 뛰어난 브라우저 기반 에디터 |
| **SSH 라이브러리 (Rust)** | **ssh2-rs** or **russh** | Rust 내부에서 SSH 세션을 열고 SFTP/SCP 프로토콜을 구현하기 위한 고성능 라이브러리 |
| **로컬 LLM 클라이언트** | **Ollama API Integration** | 로컬 백엔드나 프론트엔드에서 `http://localhost:11434` 로컬 API를 통해 손쉽게 연동 |

## 7. 단계별 개발 로드맵 (Roadmap)

### Phase 1: MVP 터미널 구현 및 한글 최적화
* Tauri 기반의 윈도우 프로젝트 세팅 및 xterm.js 연동
* 로컬 터미널 실행 환경 구축 및 한글 입력기(IME) 버그 테스트/완전 해결
* Warp 스타일의 기본적인 명령어 블록 표시 UI 구현

### Phase 2: 세션 관리 및 레이아웃 시스템
* 화면 분할(가로/세로) 및 세션 간 포커스 이동 기능 구현
* 개별 세션별 폰트 사이즈 조절 기능 (`Ctrl + +, -`)
* 세션 구성 상태(분할 구성, 세션 정보, 타이틀) 로컬 DB 저장 및 재로드 프로토콜 작성

### Phase 3: 내장 파일 관리자, 에디터 및 드래그 앤 드롭
* 로컬 및 SSH 파일 시스템을 표현하는 파일 탐색기 구현
* Monaco Editor 연동을 통한 내장 파일 수정 기능 개발
* 로컬 탐색기 <-> SSH 터미널 간 파일 드래그 앤 드롭 업로드/다운로드 로직 구현(SFTP/SCP 사용)

### Phase 4: AI 채팅 인터페이스 및 LLM 연동
* 하단 고정형 AI 채팅창 레이아웃 구성
* 에러 로그 마우스 우클릭 시 'AI 질문 컨텍스트 추가' 기능 구현
* Ollama / LM Studio 등 OpenAI 호환 규격 로컬 LLM 설정 페이지 구현 및 연동 기능 완성
* 전체 기능 테스트 및 Windows용 무설치/인스톨러 배포본 빌드
